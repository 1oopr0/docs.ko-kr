 <span data-ttu-id="55b11-130">명명 패턴을 사용하여 관련 형식을 네임스페이스로 그룹화하면 효과적으로 클래스 라이브러리를 빌드하고 문서화할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="55b11-130">The use of naming patterns to group related types into namespaces is a very useful way to build and document class libraries.</span></span> <span data-ttu-id="55b11-131">하지만 이 명명 스키마는 표시 여부, 멤버 액세스, 상속성, 보안 또는 바인딩에는 영향을 주지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="55b11-131">However, this naming scheme has no effect on visibility, member access, inheritance, security, or binding.</span></span> <span data-ttu-id="55b11-132">하나의 네임스페이스가 여러 어셈블리에 분할될 수 있으며 하나의 어셈블리에 여러 네임스페이스의 형식이 포함될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="55b11-132">A namespace can be partitioned across multiple assemblies and a single assembly can contain types from multiple namespaces.</span></span> <span data-ttu-id="55b11-133">어셈블리는 공용 언어 런타임에서 버전 관리, 배포, 보안, 로딩 및 표시 여부에 대한 형식적 구조를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="55b11-133">The assembly provides the formal structure for versioning, deployment, security, loading, and visibility in the common language runtime.</span></span>
