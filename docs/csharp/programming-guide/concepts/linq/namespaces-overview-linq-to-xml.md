## <a name="xml-names"></a><span data-ttu-id="afa17-105">XML 이름</span><span class="sxs-lookup"><span data-stu-id="afa17-105">XML Names</span></span>

<span data-ttu-id="afa17-106">XML 이름으로 인해 XML 프로그래밍이 복잡해지는 경우가 많습니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-106">XML names are often a source of complexity in XML programming.</span></span> <span data-ttu-id="afa17-107">XML 이름은 XML 네임스페이스(또는 XML 네임스페이스 URI라고 함)와 로컬 이름으로 구성되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-107">An XML name consists of an XML namespace (also called an XML namespace URI) and a local name.</span></span> <span data-ttu-id="afa17-108">XML 네임스페이스는 .NET 프로그램의 네임스페이스와 유사합니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-108">An XML namespace is similar to a namespace in a .NET program.</span></span> <span data-ttu-id="afa17-109">XML 네임스페이스를 통해 요소와 특성의 이름을 고유하게 정규화할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-109">It enables you to uniquely qualify the names of elements and attributes.</span></span> <span data-ttu-id="afa17-110">이에 따라 XML 문서의 다양한 부분 간에 이름 충돌이 방지됩니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-110">This helps avoid name conflicts between various parts of an XML document.</span></span> <span data-ttu-id="afa17-111">XML 네임스페이스를 선언한 경우 해당 네임스페이스에서만 고유해야 하는 로컬 이름을 선택할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-111">When you have declared an XML namespace, you can select a local name that only has to be unique within that namespace.</span></span>

<span data-ttu-id="afa17-112">XML 이름의 또 다른 측면은 XML *네임스페이스 접두사*입니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-112">Another aspect of XML names is XML *namespace prefixes*.</span></span> <span data-ttu-id="afa17-113">XML 접두사는 XML 이름이 복잡해지는 주된 원인입니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-113">XML prefixes cause most of the complexity of XML names.</span></span> <span data-ttu-id="afa17-114">이러한 접두사를 통해 XML 네임스페이스의 바로 가기를 만들 수 있으며 이에 따라 XML 문서가 간결하고 이해하기 쉬워집니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-114">These prefixes enable you to create a shortcut for an XML namespace, which makes the XML document more concise and understandable.</span></span> <span data-ttu-id="afa17-115">그러나 XML 접두사는 의미를 갖기 위해 컨텍스트에 의존하므로 복잡성이 가중됩니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-115">However, XML prefixes depend on their context to have meaning, which adds complexity.</span></span> <span data-ttu-id="afa17-116">예를 들어, XML 접두사 `aw`를 XML 트리의 한 부분에 있는 한 XML 네임스페이스와 연결하고 XML 트리의 다른 부분에 있는 다른 XML 네임스페이스와 연결할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-116">For example, the XML prefix `aw` could be associated with one XML namespace in one part of an XML tree, and with a different XML namespace in a different part of the XML tree.</span></span>

<span data-ttu-id="afa17-122">[!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)]에서 XML 이름을 나타내는 클래스는 <xref:System.Xml.Linq.XName>입니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-122">In [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)], the class that represents XML names is <xref:System.Xml.Linq.XName>.</span></span> <span data-ttu-id="afa17-123">XML 이름은 [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] API 전반에서 자주 나타나며, XML 이름이 필요할 때마다 <xref:System.Xml.Linq.XName> 매개 변수를 찾을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-123">XML names appear frequently throughout the [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] API, and wherever an XML name is required, you will find an <xref:System.Xml.Linq.XName> parameter.</span></span> <span data-ttu-id="afa17-124">그러나 <xref:System.Xml.Linq.XName>으로 직접 작업하는 경우는 드뭅니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-124">However, you rarely work directly with an <xref:System.Xml.Linq.XName>.</span></span> <span data-ttu-id="afa17-125"><xref:System.Xml.Linq.XName>에는 문자열에서의 암시적 변환이 포함되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="afa17-125"><xref:System.Xml.Linq.XName> contains an implicit conversion from string.</span></span>
