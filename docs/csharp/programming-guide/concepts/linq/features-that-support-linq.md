## <a name="query-expressions"></a><span data-ttu-id="a0cc0-105">쿼리 식</span><span class="sxs-lookup"><span data-stu-id="a0cc0-105">Query Expressions</span></span>

<span data-ttu-id="a0cc0-106">쿼리 식은 SQL이나 XQuery와 유사한 선언적 구문을 사용하여 IEnumerable 컬렉션을 쿼리합니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-106">Query expressions use a declarative syntax similar to SQL or XQuery to query over IEnumerable collections.</span></span> <span data-ttu-id="a0cc0-107">컴파일 시간에 쿼리 구문은 LINQ 공급자의 표준 쿼리 연산자 확장 메서드 구현에 대한 메서드 호출로 변환됩니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-107">At compile time query syntax is converted to method calls to a LINQ provider's implementation of the standard query operator extension methods.</span></span> <span data-ttu-id="a0cc0-108">애플리케이션은 `using` 지시문으로 적절한 네임스페이스를 지정하여 범위 내에 있는 표준 쿼리 연산자를 제어합니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-108">Applications control the standard query operators that are in scope by specifying the appropriate namespace with a `using` directive.</span></span> <span data-ttu-id="a0cc0-109">다음 쿼리 식은 문자열의 배열을 사용하고 문자열의 첫 번째 문자에 따라 그룹화하며 그룹의 순서를 지정합니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-109">The following query expression takes an array of strings, groups them according to the first character in the string, and orders the groups.</span></span>

```csharp
            orderby stringGroup.Key
            select stringGroup;
```

<span data-ttu-id="a0cc0-112">변수를 선언하고 초기화할 때 명시적으로 형식을 지정하는 대신 다음과 같이 [var](../../../language-reference/keywords/var.md) 한정자를 사용하여 형식을 유추하고 할당하도록 컴파일러에 지시할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-112">Instead of explicitly specifying a type when you declare and initialize a variable, you can use the [var](../../../language-reference/keywords/var.md) modifier to instruct the compiler to infer and assign the type, as shown here:</span></span>

```csharp
            where str[0] == 'm'
            select str;
```

<span data-ttu-id="a0cc0-113">`var`로 선언된 변수는 형식을 명시적으로 지정한 변수만큼 강력한 형식입니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-113">Variables declared as `var` are just as strongly typed as variables whose type you specify explicitly.</span></span> <span data-ttu-id="a0cc0-114">`var`을 사용하면 무명 형식을 만들 수 있지만 지역 변수에만 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-114">The use of `var` makes it possible to create anonymous types, but it can be used only for local variables.</span></span> <span data-ttu-id="a0cc0-115">배열은 암시적 형식 지정을 사용하여 선언할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-115">Arrays can also be declared with implicit typing.</span></span>

<span data-ttu-id="a0cc0-116">자세한 내용은 [암시적으로 형식화된 지역 변수](../../classes-and-structs/implicitly-typed-local-variables.md)를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-116">For more information, see [Implicitly Typed Local Variables](../../classes-and-structs/implicitly-typed-local-variables.md).</span></span>

## <a name="object-and-collection-initializers"></a><span data-ttu-id="a0cc0-117">개체 및 컬렉션 이니셜라이저</span><span class="sxs-lookup"><span data-stu-id="a0cc0-117">Object and Collection Initializers</span></span>

<span data-ttu-id="a0cc0-118">개체 및 컬렉션 이니셜라이저를 사용하면 개체에 대한 생성자를 명시적으로 호출하지 않고 개체를 초기화할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-118">Object and collection initializers make it possible to initialize objects without explicitly calling a constructor for the object.</span></span> <span data-ttu-id="a0cc0-119">일반적으로 이니셜라이저는 소스 데이터를 새 데이터 형식으로 프로젝션할 때 쿼리 식에서 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-119">Initializers are typically used in query expressions when they project the source data into a new data type.</span></span> <span data-ttu-id="a0cc0-120">public `Name` 및 `Phone` 속성이 있는 `Customer`라는 클래스를 가정할 경우 다음 코드에서처럼 개체 이니셜라이저를 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-120">Assuming a class named `Customer` with public `Name` and `Phone` properties, the object initializer can be used as in the following code:</span></span>

```csharp
var cust = new Customer { Name = "Mike", Phone = "555-1212" };
```

<span data-ttu-id="a0cc0-121">계속해서 `Customer` 클래스를 사용하여 `IncomingOrders`라는 데이터 소스가 있다고 가정하고 큰 `OrderSize`가 있는 각 주문의 경우 해당 주문을 기반으로 새 `Customer`를 만들려고 한다고 가정합니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-121">Continuing with our `Customer` class, assume that there is a data source called `IncomingOrders`, and that for each order with a large `OrderSize`, we would like to create a new `Customer` based off of that order.</span></span> <span data-ttu-id="a0cc0-122">LINQ 쿼리는 이 데이터 소스에서 실행하고 개체 초기화를 사용하여 컬렉션을 채울 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-122">A LINQ query can be executed on this data source and use object initialization to fill a collection:</span></span>

```csharp

                            where o.OrderSize > 5
                            select new Customer { Name = o.Name, Phone = o.Phone };
```

<span data-ttu-id="a0cc0-123">데이터 소스에는 `Customer` 클래스보다 더 많은 속성(예: `OrderSize`)이 있을 수 있지만, 개체 초기화를 사용하면 쿼리에서 반환된 데이터가 원하는 데이터 형식으로 모델링되므로 클래스와 관련된 데이터를 선택합니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-123">The data source may have more properties lying under the hood than the `Customer` class such as `OrderSize`, but with object initialization, the data returned from the query is molded into the desired data type; we choose the data that is relevant to our class.</span></span> <span data-ttu-id="a0cc0-124">따라서 이제 `IEnumerable`이 원하는 새 `Customer`로 채워집니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-124">As a result, we now have an `IEnumerable` filled with the new `Customer`s we wanted.</span></span> <span data-ttu-id="a0cc0-125">위의 내용은 LINQ 메서드 구문으로 작성할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="a0cc0-125">The above can also be written in LINQ's method syntax:</span></span>

```csharp

```
