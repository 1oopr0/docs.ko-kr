# <a name="compiler-warning-level-3-cs0675"></a><span data-ttu-id="ffdfe-103">컴파일러 경고(수준 3) CS0675</span><span class="sxs-lookup"><span data-stu-id="ffdfe-103">Compiler Warning (level 3) CS0675</span></span>
<span data-ttu-id="ffdfe-104">부호 확장 피연산자에 비트 OR 연산자를 사용했습니다. 더 작은 부호 없는 형식으로 먼저 캐스팅하세요.</span><span class="sxs-lookup"><span data-stu-id="ffdfe-104">Bitwise-or operator used on a sign-extended operand; consider casting to a smaller unsigned type first</span></span>  
  
 <span data-ttu-id="ffdfe-105">컴파일러에서 암시적으로 변수를 넓히고 부호 확장한 다음 결과 값을 비트 OR 연산에 사용했습니다.</span><span class="sxs-lookup"><span data-stu-id="ffdfe-105">The compiler implicitly widened and sign-extended a variable, and then used the resulting value in a bitwise OR operation.</span></span> <span data-ttu-id="ffdfe-106">이 경우 예기치 않은 동작이 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="ffdfe-106">This can result in unexpected behavior.</span></span>  
  
 <span data-ttu-id="ffdfe-107">다음 샘플에서는 CS0675를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="ffdfe-107">The following sample generates CS0675:</span></span>  
  
```csharp  
// CS0675.cs  
// compile with: /W:3  
using System;  
  
public class sign  
{  
   public static void Main()  
   {  
      int hi = 1;  
      int lo = 1;  
      long value = (((long)hi) << 32) | lo;              // CS0675  
      // try the following line instead  
      // long value = (((long)hi) << 32) | ((uint)lo);   // correct  
   }  
}  
```
