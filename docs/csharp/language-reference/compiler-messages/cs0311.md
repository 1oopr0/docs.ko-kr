# <a name="compiler-error-cs0311"></a><span data-ttu-id="41c3f-103">컴파일러 오류 CS0311</span><span class="sxs-lookup"><span data-stu-id="41c3f-103">Compiler Error CS0311</span></span>

<span data-ttu-id="41c3f-104">‘type1’ 형식은 제네릭 형식 또는 ‘\<name>’ 메서드에서 ‘T’ 형식 매개 변수로 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="41c3f-104">The type 'type1' cannot be used as type parameter 'T' in the generic type or method '\<name>'.</span></span> <span data-ttu-id="41c3f-105">'type1'에서 'type2'로의 암시적 참조 변환이 없습니다.</span><span class="sxs-lookup"><span data-stu-id="41c3f-105">There is no implicit reference conversion from 'type1' to 'type2'.</span></span>  
  
 <span data-ttu-id="41c3f-106">제약 조건이 제네릭 형식 매개 변수에 적용될 때는 구체적 인수에서 제약 조건 형식으로의 암시적 ID 또는 참조 변환이 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="41c3f-106">When a constraint is applied to a generic type parameter, an implicit identity or reference conversion must exist from the concrete argument to the type of the constraint.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="41c3f-107">이 오류를 해결하려면</span><span class="sxs-lookup"><span data-stu-id="41c3f-107">To correct this error</span></span>  
  
1. <span data-ttu-id="41c3f-108">클래스를 만드는 데 사용 중인 인수를 변경합니다.</span><span class="sxs-lookup"><span data-stu-id="41c3f-108">Change the argument you are using to create the class.</span></span>  
  
2. <span data-ttu-id="41c3f-109">해당 클래스를 소유하고 있다면 제약 조건을 제거하거나 다른 작업을 통해 암시적 참조 또는 ID 변환을 사용하도록 설정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="41c3f-109">If you own the class, you can remove the constraint or else do something to enable an implicit reference or identity conversion.</span></span> <span data-ttu-id="41c3f-110">예를 들어 두 번째 형식이 첫 번째 형식에서 상속을 하도록 지정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="41c3f-110">For example, you can make the second type inherit from the first.</span></span>  
  
## <a name="example"></a><span data-ttu-id="41c3f-111">예제</span><span class="sxs-lookup"><span data-stu-id="41c3f-111">Example</span></span>  
  
```csharp  
// cs0311.cs  
class B {}  
class C {}  
class Test<T> where T : C  
{ }  
  
class Program  
{  
    static void Main()  
    {  
        Test<B> test = new Test<B>(); //CS0311  
    }  
}  
```  
  
 <span data-ttu-id="41c3f-112">값 형식 인수를 사용하려고 할 때 이 오류가 발생하면 `short`에서 `int`로의 변환과 같은 암시적 숫자 변환은 제네릭 형식 매개 변수를 충족하지 않는다는 점에 유의하세요.</span><span class="sxs-lookup"><span data-stu-id="41c3f-112">If this error occurs when trying to use a value-type argument, notice that an implicit numeric conversion, for example from `short` to `int`, does not satisfy a generic type parameter.</span></span>
