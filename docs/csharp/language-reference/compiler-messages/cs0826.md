# <a name="compiler-error-cs0826"></a><span data-ttu-id="78255-103">컴파일러 오류 CS0826</span><span class="sxs-lookup"><span data-stu-id="78255-103">Compiler Error CS0826</span></span>

 <span data-ttu-id="78255-105">배열 요소는 모두 동일한 형식이거나, 컴파일러에서 사용하는 형식 유추 규칙에 따라 암시적으로 동일한 형식으로 변환 가능해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="78255-105">Array elements must all be the same type or implicitly convertible to the same type according to the type inference rules used by the compiler.</span></span> <span data-ttu-id="78255-106">가장 적합한 형식은 배열 식에 있는 형식 중 하나여야 합니다.</span><span class="sxs-lookup"><span data-stu-id="78255-106">The best type must be one of the types present in the array expression.</span></span> <span data-ttu-id="78255-107">`object` 등의 새 형식으로는 요소가 변환되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="78255-107">Elements will not be converted to a new type such as `object`.</span></span> <span data-ttu-id="78255-108">암시적으로 형식화된 배열의 경우 컴파일러에서 할당된 요소의 형식에 따라 배열 형식을 유추해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="78255-108">For an implicitly typed array, the compiler must infer the array type based on the type of elements assigned to it.</span></span>

## <a name="example"></a><span data-ttu-id="78255-113">예제</span><span class="sxs-lookup"><span data-stu-id="78255-113">Example</span></span>  

 <span data-ttu-id="78255-114">다음 코드에서는 배열 요소가 모두 동일한 형식이 아니고, 컴파일러의 형식 유추 논리를 통해 가장 적합한 형식을 찾을 수 없으므로 CS0826을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="78255-114">The following code generates CS0826 because the array elements are not all the same type, and the compiler's type inference logic does not find a single best type:</span></span>  
  
```csharp  
// cs0826.cs  
public class C  
{  
    public static int Main()  
    {  
        var x = new[] { 1, "str" }; // CS0826  
  
        char c = 'c';  
        short s1 = 0;  
        short s2 = -0;  
        short s3 = 1;  
        short s4 = -1;  
  
        var array1 = new[] { s1, s2, s3, s4, c, '1' }; // CS0826  
        return 1;  
    }  
}  
```
