# <a name="compiler-warning-level-1-cs0420"></a><span data-ttu-id="17a16-103">컴파일러 경고(수준 1) CS0420</span><span class="sxs-lookup"><span data-stu-id="17a16-103">Compiler Warning (level 1) CS0420</span></span>
<span data-ttu-id="17a16-104">‘identifier’: volatile 필드에 대한 참조는 volatile로 처리되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="17a16-104">'identifier': a reference to a volatile field will not be treated as volatile</span></span>  
  
 <span data-ttu-id="17a16-105">volatile 필드는 함수 범위 내에서 volatile로 처리되지 않으므로 일반적으로 `ref` 또는 **out** 매개 변수를 사용하여 전달해서는 안 됩니다.</span><span class="sxs-lookup"><span data-stu-id="17a16-105">A volatile field should not normally be passed using a `ref` or **out** parameter, since it will not be treated as volatile within the scope of the function.</span></span> <span data-ttu-id="17a16-106">단, interlocked API를 호출하는 경우 등은 예외입니다.</span><span class="sxs-lookup"><span data-stu-id="17a16-106">There are exceptions to this, such as when calling an interlocked API.</span></span> <span data-ttu-id="17a16-107">모든 경고와 마찬가지로, 드물긴 하지만 의도적으로 volatile 필드를 참조 매개 변수로 사용하는 경우 [#pragma warning](../preprocessor-directives/preprocessor-pragma-warning.md)을 통해 이 경고를 해제할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="17a16-107">As with any warning, you may use the [#pragma warning](../preprocessor-directives/preprocessor-pragma-warning.md) to disable this warning in those rare cases where you are intentionally using a volatile field as a reference parameter.</span></span>  
  
 <span data-ttu-id="17a16-108">다음 샘플에서는 CS0420을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="17a16-108">The following sample generates CS0420:</span></span>  
  
```csharp  
// CS0420.cs  
// compile with: /W:1  
using System;  
  
class TestClass  
{  
   private volatile int i;  
  
   public void TestVolatile(ref int ii)  
   {  
   }  
  
   public static void Main()  
   {  
      TestClass x = new TestClass();  
      x.TestVolatile(ref x.i);   // CS0420
   }  
}  
```
