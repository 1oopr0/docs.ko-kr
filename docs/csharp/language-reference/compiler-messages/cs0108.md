# <a name="compiler-warning-level-2-cs0108"></a><span data-ttu-id="7b827-103">컴파일러 경고(수준 2) CS0108</span><span class="sxs-lookup"><span data-stu-id="7b827-103">Compiler Warning (level 2) CS0108</span></span>

<span data-ttu-id="7b827-104">‘member1’은(는) 상속된 ‘member2’ 멤버를 숨깁니다.</span><span class="sxs-lookup"><span data-stu-id="7b827-104">'member1' hides inherited member 'member2'.</span></span> <span data-ttu-id="7b827-105">숨기려면 new 키워드를 사용하세요.</span><span class="sxs-lookup"><span data-stu-id="7b827-105">Use the new keyword if hiding was intended.</span></span>

 <span data-ttu-id="7b827-106">기본 클래스의 변수와 동일한 이름을 사용하여 변수가 선언되었습니다.</span><span class="sxs-lookup"><span data-stu-id="7b827-106">A variable was declared with the same name as a variable in a base class.</span></span> <span data-ttu-id="7b827-107">그러나 [new](../keywords/new-modifier.md) 키워드가 사용되지 않았습니다.</span><span class="sxs-lookup"><span data-stu-id="7b827-107">However, the [new](../keywords/new-modifier.md) keyword was not used.</span></span> <span data-ttu-id="7b827-108">이 경고는 **new**를 사용해야 한다고 알립니다. 선언에 **new**를 사용한 것처럼 변수가 선언되었습니다.</span><span class="sxs-lookup"><span data-stu-id="7b827-108">This warning informs you that you should use **new**; the variable is declared as if **new** had been used in the declaration.</span></span>

 <span data-ttu-id="7b827-109">다음 샘플에서는 CS0108을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="7b827-109">The following sample generates CS0108:</span></span>

```csharp
// CS0108.cs
// compile with: /W:2
using System;

namespace x
{
    public class clx
    {
        public int i = 1;
    }

    public class cly : clx
    {
        public static int i = 2;   // CS0108, use the new keyword
        // Use the following line instead:
        // public static new int i = 2;

      public static void Main()
      {
         Console.WriteLine(i);
      }
   }
}
```
