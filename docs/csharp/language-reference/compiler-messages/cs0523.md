<span data-ttu-id="88d13-104">‘struct1’ 형식의 ‘struct2 field’ 구조체 멤버는 구조체 레이아웃에서 순환됩니다.</span><span class="sxs-lookup"><span data-stu-id="88d13-104">Struct member 'struct2 field' of type 'struct1' causes a cycle in the struct layout</span></span>  
  
 <span data-ttu-id="88d13-105">두 구조체의 정의에 재귀 참조가 포함되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="88d13-105">The definitions of two structs include recursive references.</span></span> <span data-ttu-id="88d13-106">각 구조체가 다른 구조체에서 자신을 정의하지 않도록 [struct](../builtin-types/struct.md) 정의를 변경합니다.</span><span class="sxs-lookup"><span data-stu-id="88d13-106">Change the [struct](../builtin-types/struct.md) definitions such that each does not define itself on the other.</span></span> <span data-ttu-id="88d13-107">구조체는 값 형식이므로 이 제한은 구조체에만 적용됩니다.</span><span class="sxs-lookup"><span data-stu-id="88d13-107">This limitation applies only to structs, since structs are value types.</span></span> <span data-ttu-id="88d13-108">재귀 참조를 사용하는 경우 형식을 클래스로 선언합니다.</span><span class="sxs-lookup"><span data-stu-id="88d13-108">If using recursive references, declare your types as classes.</span></span>  
  
 <span data-ttu-id="88d13-109">다음 샘플에서는 CS0523을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="88d13-109">The following sample generates CS0523:</span></span>  
  
```csharp  
// CS0523.cs

{  
   public RecursiveLayoutStruct2 field;  
}  
  
struct RecursiveLayoutStruct2  
{  
   public RecursiveLayoutStruct1 field;   // CS0523  
}  
```
