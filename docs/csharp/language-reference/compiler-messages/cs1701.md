<span data-ttu-id="d6638-104">"Assembly Name #1" 어셈블리 참조가 "Assembly Name #2"와 일치하는 것으로 간주합니다. 런타임 정책을 지정해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="d6638-104">Assuming assembly reference "Assembly Name #1" matches "Assembly Name #2", you may need to supply runtime policy</span></span>

## <a name="example"></a><span data-ttu-id="d6638-107">예제</span><span class="sxs-lookup"><span data-stu-id="d6638-107">Example</span></span>  

 <span data-ttu-id="d6638-108">다음과 같은 다중 파일 샘플은 두 개의 서로 다른 외부 별칭을 사용하여 어셈블리를 참조합니다.</span><span class="sxs-lookup"><span data-stu-id="d6638-108">The following multifile sample references an assembly using two different external aliases.</span></span> <span data-ttu-id="d6638-109">이 첫 번째 샘플에서는 CS1701_d 어셈블리를 만드는 코드의 이전 버전을 빌드합니다.</span><span class="sxs-lookup"><span data-stu-id="d6638-109">This first sample builds the older version of the code that creates assembly CS1701_d.</span></span>  
  
```csharp  
// CS1701_a.cs  
// compile with: /target:library /out:cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("1.0")]  
public class A {  
   public void M1() {}  
}  
  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="d6638-110">예제</span><span class="sxs-lookup"><span data-stu-id="d6638-110">Example</span></span>  

 <span data-ttu-id="d6638-111">이 코드에서는 CS1701_d 어셈블리의 최신 버전을 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="d6638-111">This is the code that creates the newer version of assembly CS1701_d.</span></span> <span data-ttu-id="d6638-112">출력 파일에 같은 이름이 필요하므로 이전 버전과 다른 디렉터리에 컴파일됨에 유의하세요.</span><span class="sxs-lookup"><span data-stu-id="d6638-112">Note that it compiles into a different directory than the older version, necessary since the output files have the same names.</span></span>  
  
```csharp  
// CS1701_b.cs  
// compile with: /target:library /out:c:\\cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("2.0")]  
public class A {  
   public void M2() {}  
   public void M1() {}  
}  
  
public class C2 {}  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="d6638-113">예제</span><span class="sxs-lookup"><span data-stu-id="d6638-113">Example</span></span>  

 <span data-ttu-id="d6638-114">이 샘플은 A1 및 A2 외부 별칭을 설정합니다.</span><span class="sxs-lookup"><span data-stu-id="d6638-114">This sample sets up the external aliases A1 and A2.</span></span>  
  
```csharp  
// CS1701_c.cs  
// compile with: /target:library /reference:A2=c:\\cs1701_d.dll /reference:A1=cs1701_d.dll  
  
extern alias A1;  
extern alias A2;  
// using System;  
using a1 = A1::A;  
using a2 = A2::A;  
  
public class Ref {  
   public static a1 A1() { return new a1(); }  
   public static a2 A2() { return new a2(); }  
  
   public static A1::C1 M1() { return new A1::C1(); }  
   public static A2::C2 M2() { return new A2::C2(); }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="d6638-115">예제</span><span class="sxs-lookup"><span data-stu-id="d6638-115">Example</span></span>  

 <span data-ttu-id="d6638-116">이 샘플에서는 A의 두 개의 다른 별칭을 사용하여 메서드를 호출합니다. 다음 샘플에서는 C1701 오류가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="d6638-116">This sample calls methods using two different aliases of A. The following sample generates C1701.</span></span>

```csharp  
      Ref.A2().M2();  
   }  
}  
```
