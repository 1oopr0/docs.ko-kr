# <a name="compiler-error-cs1009"></a><span data-ttu-id="4e18b-103">컴파일러 오류 CS1009</span><span class="sxs-lookup"><span data-stu-id="4e18b-103">Compiler Error CS1009</span></span>
<span data-ttu-id="4e18b-104">인식할 수 없는 이스케이프 시퀀스입니다.</span><span class="sxs-lookup"><span data-stu-id="4e18b-104">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="4e18b-105">[string](../builtin-types/reference-types.md#the-string-type)의 백슬래시(\\) 뒤에 예기치 않은 문자가 있습니다.</span><span class="sxs-lookup"><span data-stu-id="4e18b-105">An unexpected character follows a backslash (\\) in a [string](../builtin-types/reference-types.md#the-string-type).</span></span> <span data-ttu-id="4e18b-106">올바른 이스케이프 문자 중 하나가 필요합니다.</span><span class="sxs-lookup"><span data-stu-id="4e18b-106">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="4e18b-107">자세한 내용은 [문자 이스케이프](../../../standard/base-types/character-escapes-in-regular-expressions.md)를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="4e18b-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="4e18b-108">다음 샘플에서는 CS1009를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="4e18b-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="4e18b-109">대개 파일 이름에 백슬래시 문자를 사용하는 경우 이 오류가 발생합니다. 예를 들면 다음과 같습니다.</span><span class="sxs-lookup"><span data-stu-id="4e18b-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="4e18b-110">이 오류를 해결하려면 다음 예제와 같이 "\\\\" 또는 @-quoted 문자열 리터럴을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="4e18b-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
