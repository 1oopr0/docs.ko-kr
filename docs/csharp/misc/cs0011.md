## <a name="example"></a><span data-ttu-id="df84f-109">예제</span><span class="sxs-lookup"><span data-stu-id="df84f-109">Example</span></span>  

 <span data-ttu-id="df84f-110">두 번째 파일은 이전 예제에서 만든 `C` 클래스에서 파생된 `B` 클래스를 정의하는 DLL을 만듭니다.</span><span class="sxs-lookup"><span data-stu-id="df84f-110">The second file creates a DLL that defines a class `C` that is derived from the class `B` that was created in the previous example.</span></span>

```csharp  
// compile with: /target:library /reference:CS0011_1.dll  
// post-build command: del /f CS0011_1.dll

```  
  
## <a name="example"></a><span data-ttu-id="df84f-111">예제</span><span class="sxs-lookup"><span data-stu-id="df84f-111">Example</span></span>  

 <span data-ttu-id="df84f-112">세 번째 파일은 첫 번째 단계에서 만든 DLL을 대체하고 내부 클래스 `B`의 정의를 생략합니다.</span><span class="sxs-lookup"><span data-stu-id="df84f-112">The third file replaces the DLL created by the first step, and omits the definition of the inner class `B`.</span></span>

## <a name="example"></a><span data-ttu-id="df84f-113">예제</span><span class="sxs-lookup"><span data-stu-id="df84f-113">Example</span></span>  

 <span data-ttu-id="df84f-114">마지막으로 네 번째 파일은 두 번째 예제에서 정의된 `C` 클래스( `B`클래스에서 파생되며 현재 없음)를 참조합니다.</span><span class="sxs-lookup"><span data-stu-id="df84f-114">Finally, the fourth file references the class `C` defined in the second example, which is derived from class `B`, and which is now missing.</span></span>
