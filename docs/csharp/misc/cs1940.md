<span data-ttu-id="735d4-103">소스 형식 'type'에 쿼리 패턴이 여러 번 구현되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="735d4-103">Multiple implementations of the query pattern were found for source type 'type'.</span></span> <span data-ttu-id="735d4-104">'method'에 대한 호출이 모호합니다.</span><span class="sxs-lookup"><span data-stu-id="735d4-104">Ambiguous call to 'method'.</span></span>  
  
 <span data-ttu-id="735d4-105">이 오류는 쿼리 메서드의 여러 구현이 정의되었는데 컴파일러에서 쿼리에 사용하기에 가장 적합한 구현을 구분할 수 없는 경우 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="735d4-105">This error is generated when multiple implementations of a query method are defined and the compiler cannot disambiguate which one is best to use for the query.</span></span> <span data-ttu-id="735d4-106">다음 예제에서는 두 버전 모두 하나의 `Select` 를 입력 매개 변수로 사용하고 `int` 를 반환 값으로 사용하기 때문에 `int` 의 두 버전에 동일한 서명이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="735d4-106">In the following example, both versions of `Select` have the same signature, because they both accept one `int` as an input parameter and have `int` as a return value.</span></span>

1. <span data-ttu-id="735d4-108">각 메서드에 대해 하나의 구현만 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="735d4-108">Provide only one implementation for each method.</span></span>  
  
## <a name="example"></a><span data-ttu-id="735d4-109">예제</span><span class="sxs-lookup"><span data-stu-id="735d4-109">Example</span></span>  

 <span data-ttu-id="735d4-110">다음 코드에서는 CS1940을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="735d4-110">The following code generates CS1940:</span></span>  
  
```csharp  
// cs1940.cs  
using System; //must include explicitly for types defined in 3.5  
class Test  
{  
    public delegate int Dele(int x);  
    int num = 0;  
    public int Select(Func<int, int> d)  
    {  
        return d(this.num);  
    }  
    public int Select(Dele d) // CS1940  
    {  
        return d(this.num) + 1;  
    }  
    public static void Main()  
    {  
        var q = from x in new Test()  
        select x;  
    }  
}  
```
