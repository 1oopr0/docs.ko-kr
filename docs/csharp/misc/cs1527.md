# <a name="compiler-error-cs1527"></a><span data-ttu-id="45d5b-103">컴파일러 오류 CS1527</span><span class="sxs-lookup"><span data-stu-id="45d5b-103">Compiler Error CS1527</span></span>

<span data-ttu-id="45d5b-104">네임 스페이스에 정의 된 요소는 명시적으로 private, protected, protected internal 또는 private protected로 선언할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="45d5b-104">Elements defined in a namespace cannot be explicitly declared as private, protected, protected internal or private protected.</span></span>
  
 <span data-ttu-id="45d5b-105">네임스페이스의 형식 선언에 [public](../language-reference/keywords/public.md) 또는 [internal](../language-reference/keywords/internal.md) 액세스를 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="45d5b-105">Type declarations in a namespace can have either [public](../language-reference/keywords/public.md) or [internal](../language-reference/keywords/internal.md) access.</span></span> <span data-ttu-id="45d5b-106">접근성을 지정하지 않을 경우 **internal** 이 기본값입니다.</span><span class="sxs-lookup"><span data-stu-id="45d5b-106">If no accessibility is specified, **internal** is the default.</span></span>  
  
 <span data-ttu-id="45d5b-107">다음 샘플에서는 CS1527을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="45d5b-107">The following sample generates CS1527:</span></span>  
  
```csharp  
// CS1527.cs  
namespace Sample  
{  
   private class C1 {}             // CS1527  
   protected class C2 {}           // CS1527  
   protected internal class C3 {}  // CS1527  
   private protected class C4 {}   // CS1527
}  
```  
  
 <span data-ttu-id="45d5b-108">프로그램 코드에서 네임스페이스를 명시적으로 선언하지 않을 경우 모든 형식 선언이 전역 네임스페이스 내에 암시적으로 배치되므로 다음 예제에서는 CS1527을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="45d5b-108">The following example generates CS1527 because when no namespace is explicitly declared in your program code, all type declarations are located implicitly within the global namespace.</span></span>  
  
```csharp  
//cs1527_2.cs  
using System;  
  
protected class C4 {}  
private struct S1 {}  
```
