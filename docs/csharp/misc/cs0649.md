# <a name="compiler-warning-level-4-cs0649"></a><span data-ttu-id="c1288-103">컴파일러 경고(수준 4) CS0649</span><span class="sxs-lookup"><span data-stu-id="c1288-103">Compiler Warning (level 4) CS0649</span></span>
<span data-ttu-id="c1288-104">'field' 필드에는 할당되지 않으므로 항상 기본값 'value'를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="c1288-104">Field 'field' is never assigned to, and will always have its default value 'value'</span></span>  
  
 <span data-ttu-id="c1288-105">컴파일러가 값이 할당되지 않는 초기화되지 않은 전용 또는 내부 필드 선언을 발견했습니다.</span><span class="sxs-lookup"><span data-stu-id="c1288-105">The compiler detected an uninitialized private or internal field declaration that is never assigned a value.</span></span>  
  
 <span data-ttu-id="c1288-106">다음 샘플에서는 CS0649를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="c1288-106">The following sample generates CS0649:</span></span>  
  
```csharp  
// CS0649.cs  
// compile with: /W:4  
using System.Collections;  
  
class MyClass  
{  
   Hashtable table;  // CS0649  
   // You may have intended to initialize the variable to null  
   // Hashtable table = null;  
  
   // Or you may have meant to create an object here  
   // Hashtable table = new Hashtable();  
  
   public void Func(object o, string p)  
   {  
      // Or here  
      // table = new Hashtable();  
      table[p] = o;  
   }  
  
   public static void Main()  
   {  
   }  
}  
```
