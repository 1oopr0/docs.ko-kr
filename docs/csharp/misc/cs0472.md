<span data-ttu-id="d5e3b-104">'value2' 형식의 값은 'value3' 형식의 'null'과 같을 수 없으므로 식 결과는 항상 'value1'입니다.</span><span class="sxs-lookup"><span data-stu-id="d5e3b-104">The result of the expression is always 'value1' since a value of type 'value2' is never equal to 'null' of type 'value3'</span></span>  
  
 <span data-ttu-id="d5e3b-105">상수 null 값과 함께 연산자를 사용하는 경우 컴파일러가 경고합니다.</span><span class="sxs-lookup"><span data-stu-id="d5e3b-105">The compiler should warn if you use an operator with a constant null value.</span></span>  
  
## <a name="example"></a><span data-ttu-id="d5e3b-106">예제</span><span class="sxs-lookup"><span data-stu-id="d5e3b-106">Example</span></span>  
 <span data-ttu-id="d5e3b-107">다음 샘플에서는 CS0472를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="d5e3b-107">The following sample generates CS0472.</span></span>  
  
```csharp  
public class Test  
{  
    public static int Main()  
    {  
        int i = 5;  
        int counter = 0;  
  
        // Comparison:  
        if (i == null)  // CS0472  
        // To resolve, use a valid value for i.  
            counter++;
        return counter;  
    }  
}  
```
