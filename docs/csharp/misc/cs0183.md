# <a name="compiler-warning-level-1-cs0183"></a><span data-ttu-id="ed6f3-103">컴파일러 경고(수준 1) CS0183</span><span class="sxs-lookup"><span data-stu-id="ed6f3-103">Compiler Warning (level 1) CS0183</span></span>

<span data-ttu-id="ed6f3-104">지정된 식은 항상 제공된 ('type') 형식입니다.</span><span class="sxs-lookup"><span data-stu-id="ed6f3-104">The given expression is always of the provided ('type') type</span></span>  
  
 <span data-ttu-id="ed6f3-105">조건문이 항상 **true**로 평가되는 경우 조건문이 필요하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="ed6f3-105">If a conditional statement always evaluates to **true**, then you do not need a conditional statement.</span></span> <span data-ttu-id="ed6f3-106">이 경고는 **is** 연산자를 사용하여 형식을 평가하려고 할 때 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="ed6f3-106">This warning occurs when you try to evaluate a type using the **is** operator.</span></span> <span data-ttu-id="ed6f3-107">평가가 값 형식이면 확인할 필요가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="ed6f3-107">If the evaluation is a value type, then the check is unnecessary.</span></span>  
  
 <span data-ttu-id="ed6f3-108">다음 샘플에서는 CS0183을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="ed6f3-108">The following sample generates CS0183:</span></span>  
  
```csharp  
// CS0183.cs  
// compile with: /W:1  
using System;  
public class Test  
{  
   public static void F(Int32 i32, String str)  
   {  
      if (str is Object)          // OK  
         Console.WriteLine( "str is an object" );  
      else  
         Console.WriteLine( "str is not an object" );  
  
      if (i32 is Object)   // CS0183  
         Console.WriteLine( "i32 is an object" );  
      else  
         Console.WriteLine( "i32 is not an object" ); // never reached  
   }  
  
   public static void Main()  
   {  
  
      F(0, "CS0183");  
      F(120, null);
   }  
}  
```
