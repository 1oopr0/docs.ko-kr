# <a name="compiler-warning-level-2-cs0440"></a><span data-ttu-id="111b8-103">컴파일러 경고(수준 2) CS0440</span><span class="sxs-lookup"><span data-stu-id="111b8-103">Compiler Warning (level 2) CS0440</span></span>
<span data-ttu-id="111b8-104">'global::'은 별칭이 아니라 전역 네임스페이스를 항상 참조하므로 별칭 이름을 'global'로 정의하지 않는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="111b8-104">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>  
  
 <span data-ttu-id="111b8-105">이 경고는 전역이라는 별칭을 정의할 때 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="111b8-105">This warning is issued when you define an alias named global.</span></span>  
  
## <a name="example"></a><span data-ttu-id="111b8-106">예제</span><span class="sxs-lookup"><span data-stu-id="111b8-106">Example</span></span>  
 <span data-ttu-id="111b8-107">다음 예제에서는 CS0440을 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="111b8-107">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
