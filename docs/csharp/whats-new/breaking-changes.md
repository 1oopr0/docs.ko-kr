# <a name="learn-about-any-breaking-changes-in-the-c-compiler"></a>C# 컴파일러의 호환성이 손상되는 변경에 대해 알아봅니다

[Roslyn](https://github.com/dotnet/roslyn) 팀은 C# 및 Visual Basic 컴파일러의 호환성이 손상되는 변경 목록을 유지 관리합니다. 변경 내용에 대한 정보는 GitHub 리포지토리의 다음 링크에서 확인할 수 있습니다.

- [.NET 5.0 및 C# 9.0용으로 도입되는 VS2019 버전 16.8의 호환성이 손상되는 변경](https://github.com/dotnet/roslyn/blob/master/docs/compilers/CSharp/Compiler%20Breaking%20Changes%20-%20DotNet%205.md)
- [VS2019 대비 VS2019 업데이트 1 이상의 호환성이 손상되는 변경](https://github.com/dotnet/roslyn/blob/master/docs/compilers/CSharp/Compiler%20Breaking%20Changes%20-%20post%20VS2019.md)
- [VS2017 이후 호환성이 손상되는 변경(C# 7)](https://github.com/dotnet/roslyn/blob/master/docs/compilers/CSharp/Compiler%20Breaking%20Changes%20-%20post%20VS2017.md)
- [Roslyn 2.*(VS2017)과 달라진 Roslyn 3.0(VS2019)의 호환성이 손상되는 변경](https://github.com/dotnet/roslyn/blob/master/docs/compilers/CSharp/Compiler%20Breaking%20Changes%20-%20VS2019.md)
- [Roslyn 1.*(VS2015) 및 네이티브 C# 컴파일러(VS2013 및 이전)와 달라진 Roslyn 2.0(VS2017)의 호환성이 손상되는 변경](https://github.com/dotnet/roslyn/blob/master/docs/compilers/CSharp/Compiler%20Breaking%20Changes%20-%20VS2017.md)
- [네이티브 C# 컴파일러(VS2013 및 이전)와 달라진 Roslyn 1.0(VS2015)의 호환성이 손상되는 변경](https://github.com/dotnet/roslyn/blob/master/docs/compilers/CSharp/Compiler%20Breaking%20Changes%20-%20VS2015.md)
- [C# 6의 유니코드 버전 변경](https://github.com/dotnet/roslyn/blob/master/docs/compilers/CSharp/Unicode%20Version.md)
