# <a name="entity-data-model-inheritance"></a><span data-ttu-id="be71a-102">엔터티 데이터 모델: 상속</span><span class="sxs-lookup"><span data-stu-id="be71a-102">Entity Data Model: Inheritance</span></span>

<span data-ttu-id="be71a-103">EDM (엔터티 데이터 모델)은 [엔터티 형식](entity-type.md)에 대 한 상속을 지원 합니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="be71a-104">EDM의 상속은 개체 지향 프로그래밍 언어의 클래스에 대한 상속과 유사합니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="be71a-105">개체 지향 언어의 클래스와 마찬가지로 개념적 모델에서는 다른 엔터티 형식 ( *기본 형식*)에서 상속 되는 엔터티 형식 ( *파생 된 형식*)을 정의할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="be71a-106">그러나 개체 지향 프로그래밍의 클래스와는 달리, 파생 형식은 항상 기본 형식의 모든 [속성](property.md) 및 [탐색 속성](navigation-property.md) 을 상속 합니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="be71a-107">파생 형식에서 상속된 속성을 재정의할 수는 없습니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="be71a-108">개념적 모델에서는 파생 형식이 다른 파생 형식으로부터 상속받는 상속 계층 구조를 작성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="be71a-109">계층 구조의 맨 위에 있는 형식 (파생 형식이 아닌 계층 구조에서 한 형식)을 *루트 형식*이라고 합니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="be71a-110">상속 계층 구조에서 루트 형식에 [엔터티 키](entity-key.md) 를 정의 해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="be71a-111">파생 형식이 둘 이상의 형식으로부터 상속받는 상속 계층 구조는 작성할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="be71a-112">예를 들어, `Book` 엔터티 형식이 포함된 개념적 모델에서는 각각 `FictionBook`으로부터 상속되는 파생 형식 `NonFictionBook`과 `Book`을 정의할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="be71a-113">그러나 `FictionBook` 및 `NonFictionBook` 형식에서 모두 상속되는 형식을 정의할 수는 없습니다.</span><span class="sxs-lookup"><span data-stu-id="be71a-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
