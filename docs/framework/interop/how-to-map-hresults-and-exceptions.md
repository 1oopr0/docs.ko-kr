# <a name="how-to-map-hresults-and-exceptions"></a><span data-ttu-id="dca94-104">방법: HRESULT 및 예외 매핑</span><span class="sxs-lookup"><span data-stu-id="dca94-104">How to: Map HRESULTs and Exceptions</span></span>
<span data-ttu-id="dca94-105">COM 메서드는 HRESULT를 반환하여 오류를 보고하고, .NET 메서드는 예외를 throw하여 오류를 보고합니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-105">COM methods report errors by returning HRESULTs; .NET methods report them by throwing exceptions.</span></span> <span data-ttu-id="dca94-106">런타임은 두 항목 간의 전환을 처리합니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-106">The runtime handles the transition between the two.</span></span> <span data-ttu-id="dca94-107">.NET Framework의 각 예외 클래스는 HRESULT에 매핑됩니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-107">Each exception class in the .NET Framework maps to an HRESULT.</span></span>  
  
 <span data-ttu-id="dca94-108">사용자 정의 예외 클래스는 적절한 모든 HRESULT를 지정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-108">User-defined exception classes can specify whatever HRESULT is appropriate.</span></span> <span data-ttu-id="dca94-109">이러한 예외 클래스는 예외 개체에서 **HResult** 필드를 설정하여 예외 생성 시 반환될 HRESULT를 동적으로 변경할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-109">These exception classes can dynamically change the HRESULT to be returned when the exception is generated by setting the **HResult** field on the exception object.</span></span> <span data-ttu-id="dca94-110">예외에 대한 추가 정보는 관리되지 않는 프로세스의 .NET 개체에서 구현되는 **IErrorInfo** 인터페이스를 통해 클라이언트에 제공됩니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-110">Additional information about the exception is provided to the client through the **IErrorInfo** interface, which is implemented on the .NET object in the unmanaged process.</span></span>  
  
 <span data-ttu-id="dca94-111">**System.Exception**을 확장하는 클래스를 만들 경우 생성 중에 HRESULT 필드를 설정해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-111">If you create a class that extends **System.Exception**, you must set the HRESULT field during construction.</span></span> <span data-ttu-id="dca94-112">그러지 않으면 기본 클래스는 HRESULT 값을 할당합니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-112">Otherwise, the base class assigns the HRESULT value.</span></span> <span data-ttu-id="dca94-113">예외 생성자의 값을 제공하여 기존 HRESULT에 새 예외 클래스를 매핑할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-113">You can map new exception classes to an existing HRESULT by supplying the value in the exception's constructor.</span></span>  
  
 <span data-ttu-id="dca94-114">스레드에 `IErrorInfo`가 있는 경우 런타임이 `HRESULT`를 무시할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-114">Note that the runtime will sometimes ignore an `HRESULT` in cases where there is an `IErrorInfo` present on the thread.</span></span>  <span data-ttu-id="dca94-115">`HRESULT` 및 `IErrorInfo`가 동일한 오류를 나타내지 않을 경우 이 동작이 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dca94-115">This behavior can occur in cases where the `HRESULT` and the `IErrorInfo` do not represent the same error.</span></span>
