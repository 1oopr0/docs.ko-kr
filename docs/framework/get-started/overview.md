## <a name="features-of-the-common-language-runtime"></a><span data-ttu-id="f1c88-130">공용 언어 런타임의 기능</span><span class="sxs-lookup"><span data-stu-id="f1c88-130">Features of the common language runtime</span></span>

<span data-ttu-id="f1c88-131">공용 언어 런타임에서는 메모리, 스레드 실행, 코드 실행, 코드 안전성 확인, 컴파일 및 기타 시스템 서비스를 관리합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-131">The common language runtime manages memory, thread execution, code execution, code safety verification, compilation, and other system services.</span></span> <span data-ttu-id="f1c88-132">이러한 기능은 공용 언어 런타임에서 실행되는 관리 코드의 기본 기능입니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-132">These features are intrinsic to the managed code that runs on the common language runtime.</span></span>

<span data-ttu-id="f1c88-133">보안과 관련하여 관리되는 구성 요소에는 인터넷, 기업 네트워크 및 로컬 컴퓨터 등 해당 출처를 비롯한 여러 요소에 따라 다양한 신뢰도가 부여됩니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-133">Regarding security, managed components are awarded varying degrees of trust, depending on a number of factors that include their origin (such as the Internet, enterprise network, or local computer).</span></span> <span data-ttu-id="f1c88-134">이는 관리되는 구성 요소가 동일한 활성 앱에 사용되는 경우에도 파일 액세스 작업, 레지스트리 액세스 작업 또는 기타 중요한 기능을 수행할 수 있을 수도 있고 수행할 수 없을 수도 있음을 의미합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-134">This means that a managed component might or might not be able to perform file-access operations, registry-access operations, or other sensitive functions, even if it's used in the same active app.</span></span>

<span data-ttu-id="f1c88-135">또한 런타임에서는 CTS(공용 형식 시스템)라고 하는 엄격한 형식 및 코드 확인 인프라를 구현하여 코드 견고성을 유지합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-135">The runtime also enforces code robustness by implementing a strict type-and-code-verification infrastructure called the common type system (CTS).</span></span> <span data-ttu-id="f1c88-136">CTS는 모든 관리 코드가 자체 설명 방식이 되도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-136">The CTS ensures that all managed code is self-describing.</span></span> <span data-ttu-id="f1c88-137">Microsoft와 타사의 다양한 언어 컴파일러에서는 CTS에 맞는 관리 코드를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-137">The various Microsoft and third-party language compilers generate managed code that conforms to the CTS.</span></span> <span data-ttu-id="f1c88-138">따라서 관리 코드에서는 형식 준수성 및 형식 안전성을 엄격하게 유지하면서 다른 관리되는 형식 및 인스턴스를 소비할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-138">This means that managed code can consume other managed types and instances, while strictly enforcing type fidelity and type safety.</span></span>

<span data-ttu-id="f1c88-139">또한 런타임의 관리되는 환경에서는 일반적인 소프트웨어 문제를 대부분 제거합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-139">In addition, the managed environment of the runtime eliminates many common software issues.</span></span> <span data-ttu-id="f1c88-140">예를 들어, 런타임에서는 개체가 더 이상 사용되지 않을 때 해당 개체를 해제하면서 자동으로 개체 레이아웃을 처리하고 개체에 대한 참조를 관리합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-140">For example, the runtime automatically handles object layout and manages references to objects, releasing them when they are no longer being used.</span></span> <span data-ttu-id="f1c88-141">이 자동 메모리 관리 기능은 앱의 두 가지 일반적인 오류인 메모리 누수와 잘못된 메모리 참조 문제를 해결합니다.</span><span class="sxs-lookup"><span data-stu-id="f1c88-141">This automatic memory management resolves the two most common app errors, memory leaks and invalid memory references.</span></span>
