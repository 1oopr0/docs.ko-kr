<span data-ttu-id="e9d16-103">'Equals' 연산자의 양쪽에서 범위 변수를 하나 이상 참조해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="e9d16-103">You must reference at least one range variable on both sides of the 'Equals' operator.</span></span> <span data-ttu-id="e9d16-104">범위 변수는 \<variable(s)> ' Equals ' 연산자의 한 쪽에 표시 되어야 하며 범위 변수 \<variable(s)> 는 다른 쪽에 표시 되어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="e9d16-104">Range variable(s) \<variable(s)> must appear on one side of the 'Equals' operator, and range variable(s) \<variable(s)> must appear on the other.</span></span>  
  
 <span data-ttu-id="e9d16-105">LINQ 쿼리에 조인할 컬렉션에 대해 식별된 범위 변수는 식별된 컬렉션에 따라 `Equals` 연산자의 반대쪽에 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="e9d16-105">Range variables identified for collections to be joined in a LINQ query must be on opposite sides of the `Equals` operator, depending on the collection they are identified for.</span></span> <span data-ttu-id="e9d16-106">즉, 조인 중인 컬렉션 중 하나에 대해 식별된 범위 변수는 조인 중인 다른 컬렉션의 범위 변수에 대해 `Equals` 연산자의 반대쪽에 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="e9d16-106">That is, range variables identified for one of the collections being joined must be on the opposite side of the `Equals` operator from range variables for the other collection being joined.</span></span> <span data-ttu-id="e9d16-107">각 컬렉션의 범위 변수를 `Equals` 연산자의 같은 쪽에 함께 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="e9d16-107">Range variables from separate collections cannot be mixed on the same side of the `Equals` operator.</span></span>  
  
 <span data-ttu-id="e9d16-108">조인 중인 각 컬렉션의 변수 중 적어도 하나는 `Equals` 연산자의 양쪽에서 참조되어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="e9d16-108">At least one variable from each collection being joined must be referenced on each side of the `Equals` operator.</span></span>
